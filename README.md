# ETL Pipeline

This repository provisions a fully automated, serverless ETL pipeline on AWS using Terraform. It ingests raw CSV files from S3, validates them with Lambda, transforms data through AWS Glue, and outputs optimized Parquet datasets for downstream analytics. The entire workflow is instrumented with CloudWatch and surfaced in Grafana for unified observability across all serverless components.

---

## Architecture Overview

### S3 Buckets
- **Raw bucket**: Stores incoming CSV files. Uploading a new object triggers the Lambda function.
- **Processed bucket**: Stores transformed Parquet files generated by the Glue job.
- **Scripts bucket**: Stores the Glue ETL script (`transform.py`).
- **Athena results bucket**: Stores Athena query results.

### Lambda Function
- Validates incoming files and triggers the Glue job.
- Runtime: Python 3.12.
- Packaged as a ZIP file generated by Terraform.
- Environment variable provides the Glue job name.
- IAM role allows S3 access and Glue job execution.

### Glue Job
- Executes the ETL transformation logic defined in `transform.py`.
- Reads CSV files from the raw bucket.
- Writes Parquet output to the processed bucket.
- Runs on Glue 4.0 using G.1X workers.
- Uses a Glue catalog database for metadata.

### IAM Roles and Policies
- **Glue role**: Grants Glue access to S3 and attaches the AWSGlueServiceRole.
- **Lambda role**: Grants Lambda basic execution, S3 access, and Glue StartJobRun permissions.
- Inline and managed policies are attached automatically.

### S3 Event Trigger
- The raw bucket is configured to invoke the Lambda function on `s3:ObjectCreated:*` events.
- This enables fully automated ingestion and processing.

---

## End-to-End Flow

1. A CSV file is uploaded to the raw S3 bucket.
2. S3 triggers the Lambda validator.
3. Lambda starts the Glue ETL job.
4. Glue reads the raw CSV, transforms it, and writes Parquet output to the processed bucket.
5. Athena can query the processed data using the Glue catalog.

---

## Outputs

Terraform provides the following outputs:

- Glue job name
- Lambda function name
- Raw bucket name
- Processed bucket name
